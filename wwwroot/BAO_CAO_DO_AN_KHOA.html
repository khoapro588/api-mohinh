1. GIỚI THIỆU DỰ ÁN

Dự án ModelStore là một nền tảng thương mại điện tử chuyên cung cấp các sản phẩm mô hình, figure cao cấp. Hệ thống được xây dựng theo mô hình Monolithic (Khối thống nhất) nhằm đảm bảo hiệu suất cao và dễ quản lý.

2. CÔNG NGHỆ SỬ DỤNG

Thành phần Công nghệ sử dụng
Back-end ASP.NET Core Web API 9.0 (C#)
Cơ sở dữ liệu MySQL Server (thông qua Entity Framework Core)
Front-end HTML5, CSS3 (Vanilla), JavaScript (ES6+)
Thanh toán VNPay API Integration
Báo cáo/Biểu đồ Chart.js API

3. CÁC BƯỚC TRIỂN KHAI CHI TIẾT

Bước 1: Thiết lập cấu trúc Project Back-end

Khởi tạo project ASP.NET Core API. Cấu hình Entity Framework Core để kết nối với MySQL. Thiết lập cơ chế tự động khởi tạo Database (Auto-Migration) để hệ thống tự tạo bảng khi chạy lần đầu.

Bước 2: Xây dựng Cơ sở dữ liệu (Database Schema)

Thiết kế các bảng chính:

  • FigureModels: Quản lý thông tin sản phẩm (Tên, giá, ảnh, số lượng kho).
  • Orders & OrderItems: Quản lý đơn hàng và chi tiết sản phẩm khách mua.
  • Customers: Lưu trữ thông tin khách hàng và phân quyền (Admin/Customer).
  • Reviews: Hệ thống đánh giá từ người dùng.
Bước 3: Phát triển hệ thống API Controllers

Viết các Controller xử lý logic nghiệp vụ:

  • ModelsController: CRUD sản phẩm.
  • OrdersController: Xử lý đặt hàng, tính tổng tiền và thống kê doanh thu.
  • AccountsController: Đăng ký, đăng nhập và phân quyền JWT.
Bước 4: Xây dựng Giao diện Người dùng (Storefront)

Thiết kế giao diện hiện đại với phong cách "Premium Dark Mode":

  • Trang chủ (index.html): Hiển thị sản phẩm nổi bật, banner động.
  • Chi tiết sản phẩm: Xem thông tin, chọn số lượng và thêm vào giỏ hàng.
  • Giỏ hàng & Thanh toán: Xử lý logic giỏ hàng tại LocalStorage và tích hợp thanh toán VNPay.
Bước 5: Xây dựng Dashboard Quản trị (Admin Dashboard)

Tạo trang admin.html dành riêng cho người quản lý:

  • Thống kê tổng doanh thu, số lượng đơn hàng, khách hàng mới.
  • Biểu đồ doanh thu: Sử dụng Chart.js để vẽ biểu đồ tăng trưởng theo ngày/tháng.
  • Quản lý trạng thái đơn hàng (Chờ duyệt -> Đang giao -> Đã giao).
Bước 6: Tối ưu hóa và Đồng bộ dữ liệu (Fixing Bugs)

Đây là bước quan trọng nhất để hoàn thiện dự án:

  • Cấu hình camelCase JSON Serialization trong Program.cs để Front-end (JS) và Back-end (C#) hiểu nhau.
  • Xử lý lỗi real-time update: Khi khách đặt hàng, dashboard Admin sẽ cập nhật số liệu ngay lập tức.
  • Kích hoạt Swagger/Scalar UI để làm tài liệu hướng dẫn sử dụng API chuyên nghiệp.

4. CÁC TÍNH NĂNG NỔI BẬT ĐÃ HOÀN THÀNH
